语法

标题（headline）

以 * 开头的行，标题的层级由 * 的数量决定：

* 标题一
** 标题二
*** 标题三

条目（Entry）

标题及其内容成为条目（Entry），条目包括：

  • 段落
  • 列表
  • 抽屉（DRAWER）

正文/段落（paragraph）

正文紧跟着标题的下一行书写， 首行空格（ leading-space ） 解析的时候会被忽略，我偏向于不使用首行空格。一个换行符会被渲染成一个空格， \\ 是强制换行。

org 也可以开启自动缩进（不使用），默认缩进为标题的层级 +1。

(setq org-adapt-indentation t)

org-indent-mode 可以让 org 的结构看起来有缩进，但实际并没有空格，比如 gtd 文件，就是启用了这个模式。也可以通过 startup 属性 indent 配置。

列表

  • TAB (org-cycle) 折叠子列表，如果列表项还没有文本，则切换列表缩进
  • M-<RET> 插入新的列表项
  • M-S-<RET> 插入新的 checkbox 列表
  • S-<UP>/<DOWN> 同级列表项间移动
  • M-<Up>/<DOWN> 移动列表项
  • M-<LEFT>/<RIGHT> 切换列表项缩进
  • M-S-<LEFT>/<RIGHT> 切换列表项及其子列表缩进
  • 有效的列表标识 : -, +, *, 1., 1)
  • C-c ^ 列表排序

块（Blocks）

块的格式如下：

#+begin_TYPE
CONTENT
#+end_TYPE
  • 在块中显示块语法，需要转义 #+ -> ,#+
  • 显示标题，需要转义 * -> ,*
  • C-c C-, 快速插入内置的块类型

兼容性：

  • org-ruby 忽略自定义 type
  • pandoc 会解析自定义 type
  • verse 块，保留换行格式
  • quote，引用

    Everything should be made as simple as possible, but not any simpler —Albert Einstein

  • center，置中

    Everything should be made as simple as possible,
    but not any simpler

Literal Examples

type 为 example 或 src 的块，称为 literal examples. 支持块内跳转：

Here is example (ref:sc)
[[(sc)][跳转到块内]]

跳转到块内 org-ruby、pandoc 都不支持

example 支持单行语法

标记为 ~: ~

: * Headline

Drawers

C-c C-x d (org-insert-drawer) 用于描述条目的元数据，属于不可见的数据。

  • org-ruby 会解析显示出 drawer 的内容
  • pandoc 会解析显示出 drawer 的内容

[#B] 属性（Property）

条目的属性需要在 headline 或 schedule line 的下一行中定义，属性 drawer 不应出现空行。

:PROPERTIES:
:KEY:      value
:KEY_ALL: 为子项定义词表_
:END:
#+PROPERTY:

也可以用一行属性

#+PROPERTY: KEY VALUE
  • C-c C-x p ( org-set-property ) 可以为当前条目设置属性
  • org 的特殊属性

表格

org 的表格语法看起来与 markdown 一致

NamePhoneAge
Peter123417
Anna432125
| Name  | Phone | Age |
|-------+-------+-----|
| Peter |  1234 |  17 |
| Anna  |  4321 |  25 |
  • <TAB> 加上行的一个非空白字符为 |, |- 几乎可以生成整个表格
  • C-c | 可以当成 dwim 的方式创建表格
  • C-c C-c 对齐表格， <TAB> 在有文本的单元格内可以对齐表格

链接

  • DESCRIPTION
  • https://google.com
  • \test\
  • [[\[\]][[]​]]
- [[https://google.com][DESCRIPTION]]
- [[https://google.com]]
- [[\test\\][\test\]]
- [[\[\]][[]​]]
  • org-ruby 不能正确解析， link 转义 [[\[\]][[]​]] 不能被解析为 []
  • pandoc 不能解析 [[\test\\][\test\]]

内部链接，仅 pandoc 支持 custom id.

  • <<my-target>>
  • <<<radio-target>>>
  • my-target
  • radio-target
  • *math
  • name-target
  • #my-custom-id

另外还有一个实用操作：

  • org-store-link 将当前位置的链接压入栈
  • org-insert-link 可将栈中的链接弹出，插入当前位置

图片

图片的语法等同于不带描述的链接格式，支持附加元数据，通过后缀名判断是否是图片

  • org 不支持外部图片
  • org-ruby 支持 web 图片，不支持元数据
  • pandoc 支持 web 图片。

#+CAPTION: This is the caption
#+NAME:   google
[[https://www.google.com/images/branding/googlelogo/1x/googlelogo_light_color_272x92dp.png]]

另外，包括图片、表格、块都支持通过元数据设置标题（captions）

#+CAPTION: This is caption

附件

默认的附件行为会在 org 文件同目录下的 data(org-attach-id-dir) 文件夹生成 ID 文件夹，一个条目对于一个 ID。
  • org-ruby 不支持 attachment link
  • pandoc 不支持 attachment link

强调（ Emphasis ）

默认的强调语法，需要确保包围字符与外部字符之间有空格隔开，对于中文来说水土不服1。解决的方向有两个：

  1. 修改 org-emphasis-regexp-components，改变量控制对强调语法的匹配
  2. 自动加空格或零宽空格（~C-x 8 RET~），比如 ingtshan/separate-inline.el 或 zk-phi/electric-spacing

考虑源自 markdown 的使用习惯和一致性，我选择修改 org-emphasis-regexp-components2.

(org-set-emph-re 'org-emphasis-regexp-components
 (let ((cjk "[:nonascii:]")) ;; 应该使用 \\cc\\cj\\ch 但 char alternative 不支持 category 使用只能用 char class.
   (pcase-let ((`(,f ,s . ,r) org-emphasis-regexp-components))
     `(,(concat f cjk) ,(concat s cjk) . ,r)
     )
   ))

org-ruby 这块的正则是写死，需要改一下代码。ruby 的 character alternatives(character classes) 是支持 \ 转义的，可以使用 \p{Han} 这样的类别字符集，但是匹配全角符号和零宽空格麻烦一些3。

最终效果如下：

粗体（ bold ）、斜体（ italic ）、下划线（ underlined ）、抄录？（ verbatim ）、代码（ code ）、删除线（ strike-through ）

*粗体（ bold ）*、/斜体（ italic ）/、_下划线（ underlined ）_、=抄录？（ verbatim ）=、~代码（ code ）~、+删除线（ strike-through ）+

code 和 verbatim 内的字符串不被当作 org 语法处理。

脚注

这是一个脚注链接4， C-c C-c 可以创建脚注。脚注自动创建在最后一个名为 Footnotes 的条目里。脚注语法如下：

这是脚注[fn:1]
[fn:1] 这是脚注定义

org-ruby 可支持行内脚注name但不支持匿名。行内脚注的定位会输出在当前条目后的 Footnotes 条目内，但该行为存在严重 bug: 脚注定义会在声明后的每个条目内重复生成5。另外不管当前条目处于什么层级，Footnotes 条目都固定为 h2, 总体问题较多，好在使用场景较少就不一一修复了。

[fn:name: 这是一个名为 name 的行内脚注]
[fn:: 匿名脚注]

行内脚注也支持脚注定义，用于补充引用。

org-ruby 默认不开启脚注支持，需要在属性中打开支持：

#+OPTIONS: f:t

pandoc 不支持命名脚注

Footnotes:

上标下标及特殊符号

The radius of the sun is R_sun = 6.96 x 10^8 m. On the other hand, the radius of Alpha Centauri is RAlpha Centauri = 1.28 x Rsun.

The radius of the sun is R_sun = 6.96 x 10^8 m.  On the other hand, the radius of Alpha Centauri is R_{Alpha Centauri} = 1.28 x R_{sun}.

一些内置特殊符号：

  • α
  • π{d}
  • →
  • 见变量 org-entitiens
  • C-c C-x \ (org-toggle-pretty-entities) 切换文本显示为 WYSIWYM
  • org-ruby 只支持 _{}, ^{}

LaTex

行内 LaTex 标记：

If $a^2=b$ and \( b=2 \), then the solution must be either $$ a=+\sqrt{2} $$ or \[ a=-\sqrt{2} \].

能自动识别 Latex 块，无需特殊包围标记：

\begin{align*} & φ(x,y) = φ \left(∑i=1^n x_ie_i, ∑j=1^n y_je_j \right) = ∑i=1^n ∑j=1^n x_i y_j φ(e_i, e_j) =
& (x_1, \ldots, x_n) \left( \begin{array}{ccc} φ(e_1, e_1) & \cdots & φ(e_1, e_n) \ \vdots & \ddots & \vdots \ φ(e_n, e_1) & \cdots & φ(e_n, e_n) \end{array} \right) \left( \begin{array}{c} y_1 \ \vdots \ y_n \end{array} \right) \end{align*}

org-ruby

  • $$ 不支持
  • \( \) 支持
  • $$$$, \[\] 当成块处理
  • 基本不支持自动识别

pandoc

  • $$ 支持
  • \( \) 支持
  • $$$$, \[\] 不支持
  • 不支持自动识别
